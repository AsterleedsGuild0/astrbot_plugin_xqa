# AstrBot XQA 插件 PRD

## 1. 背景

AstrBot 生态已有关键词回复类插件，但主流交互通常是 `/添加检测词 <关键词> <回复>` 这类命令式格式。YuiChyanBot / XQA 提供了更符合群聊自然表达的“问答记忆”交互：

```text
我问XXX你答XXX
有人问XXX你答XXX
```

本项目希望为 AstrBot 实现一个独立的 XQA 插件，保留 XQA 的核心体验：用户可以在群聊中用自然语言设置问答，后续群消息命中问题时，Bot 自动回复对应答案。

参考实现：

- GitHub：<https://github.com/azmiao/YuiChyanBot>

本地参考路径、实现调研记录等开发环境相关信息不写入 PRD，由项目本地 `.opencode/` 目录维护。

---

## 2. 产品目标

- 提供 AstrBot 可用的 XQA 风格问答插件。
- 支持自然语言设置：`我问A你答B`、`有人问A你答B`。
- 支持群聊自动回复：用户发送问题 A，Bot 回复 B。
- 支持个人问答与群公共问答两类作用域。
- 支持正则问题、回流参数、随机回答等 XQA 核心能力。
- 提供基础管理能力：查看、搜索、删除、启停个人问答。
- 兼容 AstrBot 插件体系，尽量少依赖外部服务。

---

## 3. 非目标

- 不做大而全的通用关键词回复管理平台。
- 首版不强制实现完整 WebUI / Pages 管理界面。
- 首版不实现跨 Bot、跨 AstrBot 实例的数据同步。
- 首版不保证完全兼容 HoshinoBot XQA 的历史数据格式，但应预留导入能力。
- 首版不实现复杂的权限系统，只映射 AstrBot 可获得的管理员、超级管理员、普通成员权限。

---

## 4. 用户角色

### 4.1 普通群成员

- 可以设置自己的个人问答。
- 可以查看、搜索、删除自己的个人问答。
- 在管理员允许时，个人问答可被自己触发。

### 4.2 群管理员 / 群主

- 可以设置本群公共问答。
- 可以查看、搜索、删除本群公共问答。
- 可以查看或删除指定成员的个人问答。

### 4.3 Bot 超级管理员

- 可以启用或禁用某群个人问答功能。
- 可以清空某群个人问答或公共问答。
- 可选支持全群问答、数据导入导出、跨群复制等高级维护能力。

## 5. 核心概念

### 5.1 个人问答

由用户通过 `我问A你答B` 设置，仅该用户在同一群内发送 A 时触发。

示例：

```text
我问菜单你答今天吃黄焖鸡
```

触发：

```text
菜单
```

Bot 回复：

```text
今天吃黄焖鸡
```

### 5.2 公共问答

由群管理员通过 `有人问A你答B` 设置，本群任意用户发送 A 时触发。

示例：

```text
有人问群规你答禁止刷屏，禁止攻击他人
```

### 5.3 全群问答

维护能力。由超级管理员通过 `全群问A你答B` 设置，将公共问答写入 Bot 所在多个群。

首版可以作为 P1 或 P2 能力，不强制进入 MVP。

## 6. 功能需求

### 6.1 设置问答

#### 6.1.1 个人问答

命令格式：

```text
我问<问题>你答<回答>
```

要求：

- 仅群聊生效。
- `<问题>` 与 `<回答>` 均不能为空。
- 普通成员可使用。
- 如果当前群禁用了个人问答，普通成员设置应失败或静默忽略，建议返回明确提示。
- 如果同一用户在同一群重复设置相同问题，应覆盖旧回答。

#### 6.1.2 公共问答

命令格式：

```text
有人问<问题>你答<回答>
```

要求：

- 仅群聊生效。
- 需要群管理员、群主或 Bot 超级管理员权限。
- 同一群重复设置相同问题时覆盖旧回答。

#### 6.1.3 全群问答

命令格式：

```text
全群问<问题>你答<回答>
```

要求：

- 需要 Bot 超级管理员权限。
- 写入范围为 Bot 当前可识别的所有群，或配置中允许的群。
- 首版可暂不实现，但 PRD 预留。

### 6.2 自动回复

监听群消息并匹配问答。

匹配顺序：

1. 当前群、当前用户的个人问答。
2. 当前群公共问答。

命中后发送答案。若个人问答已命中，不再匹配公共问答。

### 6.3 匹配规则

#### 6.3.1 完全匹配优先

如果用户消息与问题文本完全一致，优先命中。

#### 6.3.2 正则匹配

如果完全匹配失败，将问题作为正则表达式匹配。

建议匹配方式：

```regex
^<问题>$
```

要求：

- 后设置的问题优先匹配。
- 正则异常时跳过该问题，不影响其他问答。
- 应增加泛匹配/危险正则限制，避免 `.*`、空匹配、灾难性回溯导致刷屏或性能问题。

#### 6.3.3 回流参数

回答支持 `$1`、`$2` 等占位符，对应正则捕获组。

示例：

```text
我问你好(.*)你答你也好$1
```

用户发送：

```text
你好呀
```

Bot 回复：

```text
你也好呀
```

### 6.4 随机回答

回答支持使用 `#` 分隔多个候选答案。

示例：

```text
有人问吃什么你答火锅#烧烤#黄焖鸡
```

命中后随机回复其中一条。

要求：

- `\#` 表示普通 `#` 字符，不参与分割。
- 存储时应保存为答案列表。

### 6.5 查看与搜索

#### 6.5.1 查看个人问答

命令：

```text
看看我问
看看我问<搜索词>
```

返回当前群当前用户设置的问题列表。带搜索词时进行包含搜索。

#### 6.5.2 查看公共问答

命令：

```text
看看有人问
看看有人问<搜索词>
```

返回当前群公共问答问题列表。

#### 6.5.3 查看指定成员问答

命令：

```text
查问答 @某人
查问答 @某人 <搜索词>
```

要求：

- 需要群管理员、群主或 Bot 超级管理员权限。
- 返回指定成员在当前群设置的个人问答列表。

### 6.6 删除问答

#### 6.6.1 删除自己的问答

命令：

```text
不要回答<问题>
```

行为：

- 普通成员优先删除自己设置的个人问答。
- 如果没有自己的个人问答，但存在公共问答，普通成员不能删除公共问答。
- 管理员可删除公共问答。

#### 6.6.2 管理员删除指定成员问答

命令：

```text
@某人 不要回答<问题>
```

要求：

- 需要群管理员、群主或 Bot 超级管理员权限。

#### 6.6.3 删除全群问答

命令：

```text
全群不要回答<问题>
```

要求：

- 需要 Bot 超级管理员权限。
- 首版可作为 P1 / P2 能力。

### 6.7 群级开关

命令：

```text
XQA禁用我问
XQA启用我问
```

要求：

- 需要 Bot 超级管理员权限，或配置项允许群管理员操作。
- 禁用后：
  - 普通成员不能设置 `我问`。
  - 自动回复时不匹配个人问答。
  - 查看个人问答可返回禁用提示。
- 公共问答不受影响。

### 6.8 清空能力

命令：

```text
XQA清空本群所有我问
XQA清空本群所有有人问
```

要求：

- 需要 Bot 超级管理员权限。
- 操作前建议二次确认，或至少在配置项中支持禁用高危命令。

### 6.9 帮助

命令：

```text
问答帮助
XQA帮助
```

返回插件说明、命令列表、正则与随机回答示例。

## 7. 数据需求

### 7.1 推荐数据结构

逻辑结构：

```json
{
  "config": {
    "<group_id>": {
      "self_enabled": true
    }
  },
  "groups": {
    "<group_id>": {
      "all": {
        "<question>": ["<answer_1>", "<answer_2>"]
      },
      "users": {
        "<user_id>": {
          "<question>": ["<answer_1>", "<answer_2>"]
        }
      }
    }
  }
}
```

实现时可使用 SQLite、JSON、AstrBot 插件数据目录中的文件存储，或轻量 KV 存储。

### 7.2 持久化要求

- 数据应存放在 AstrBot 插件推荐的数据目录中，避免写入源码目录。
- 插件重启后问答仍可用。
- 写入应尽量原子化，避免进程退出导致数据损坏。
- 数据文件应避免进入 Git。

### 7.3 排序要求

- 查看列表时默认按创建时间或设置顺序展示。
- 匹配时后设置的问题优先，避免旧泛匹配抢占新规则。

## 9. 配置项需求

建议提供以下配置项：

```yaml
self_question_enabled_default: true
allow_group_admin_toggle_self_question: false
allow_group_admin_clear_questions: false
enable_regex_question: true
max_question_length: 200
max_answer_length: 1000
max_answers_per_question: 20
max_questions_per_user_per_group: 100
max_public_questions_per_group: 300
reject_empty_regex: true
reject_dangerous_regex: true
match_timeout_ms: 50
quote_reply: false
cooldown_seconds: 0
enable_global_question: false
enable_image_message: true
persist_image_as_base64: true
max_images_per_answer: 5
enable_processing_feedback: true
processing_emoji_ids:
  - "424"
  - "66"
enable_data_export: false
enable_data_import: false
storage_backend: json
storage_filename: xqa_data.json
list_page_size: 30
command_prefix_required: false
permission_denied_notice: true
```

配置项说明：

| 配置项 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `self_question_enabled_default` | `bool` | `true` | 新群默认是否启用个人问答。关闭后，普通成员不能使用 `我问A你答B`，自动回复也不会匹配个人问答。 |
| `allow_group_admin_toggle_self_question` | `bool` | `false` | 是否允许群管理员使用 `XQA启用我问` / `XQA禁用我问`。关闭时仅 Bot 超级管理员可操作。 |
| `allow_group_admin_clear_questions` | `bool` | `false` | 是否允许群管理员清空本群问答。默认关闭，避免误删大量数据。 |
| `enable_regex_question` | `bool` | `true` | 是否允许问题文本作为正则表达式参与匹配。关闭后仅支持完全匹配。 |
| `max_question_length` | `int` | `200` | 单个问题最大字符数。用于限制超长问题和异常正则。 |
| `max_answer_length` | `int` | `1000` | 单条原始回答最大字符数，按用户输入的完整回答计算。 |
| `max_answers_per_question` | `int` | `20` | 单个问题最多允许的随机回答数量，即 `#` 分割后的候选答案上限。 |
| `max_questions_per_user_per_group` | `int` | `100` | 单个用户在单个群内最多可设置的个人问答数量。 |
| `max_public_questions_per_group` | `int` | `300` | 单个群最多可设置的公共问答数量。 |
| `reject_empty_regex` | `bool` | `true` | 是否拒绝可匹配空字符串或明显泛匹配的问题，例如空问题、`.*` 等。 |
| `reject_dangerous_regex` | `bool` | `true` | 是否启用危险正则拦截。首版可先实现保守规则，后续再引入更完整的 ReDoS 检测。 |
| `match_timeout_ms` | `int` | `50` | 单条正则匹配的目标超时时间。若使用 Python 标准 `re` 无法直接超时，应在实现说明中注明采用保守检测或第三方库。 |
| `quote_reply` | `bool` | `false` | 自动回复时是否引用触发消息。 |
| `cooldown_seconds` | `int` | `0` | 同群或同规则触发冷却时间。0 表示不启用冷却。 |
| `enable_global_question` | `bool` | `false` | 是否启用 `全群问A你答B`、`全群不要回答A` 等跨群维护命令。 |
| `enable_image_message` | `bool` | `true` | 是否支持图片回答以及文本 + 图片复合回答。 |
| `persist_image_as_base64` | `bool` | `true` | 是否尽量将图片转为 base64 持久化保存，降低临时图片 URL 过期风险。 |
| `max_images_per_answer` | `int` | `5` | 单条回答最多允许保存的图片数量。 |
| `enable_processing_feedback` | `bool` | `true` | 图片保存耗时较长时是否发送处理反馈。优先使用 QQ 表情回应，失败时发送文本提示。 |
| `processing_emoji_ids` | `list[string]` | `["424", "66"]` | OneBot/NapCat `set_msg_emoji_like` 使用的表情 ID 候选列表。 |
| `enable_data_export` | `bool` | `false` | 是否启用数据导出命令。属于维护功能，默认关闭。 |
| `enable_data_import` | `bool` | `false` | 是否启用数据导入或重建命令。属于高危维护功能，默认关闭。 |
| `storage_backend` | `string` | `json` | 数据存储后端。首版建议支持 `json`，后续可扩展为 `sqlite`。 |
| `storage_filename` | `string` | `xqa_data.json` | 数据文件名。文件应位于 AstrBot 插件数据目录，不应写入源码目录。 |
| `list_page_size` | `int` | `30` | 查看问答列表时单次返回的问题数量，用于避免刷屏。 |
| `command_prefix_required` | `bool` | `false` | 自然语言命令是否必须带 AstrBot 唤醒前缀。默认保持 XQA 原始体验，不要求前缀。 |
| `permission_denied_notice` | `bool` | `true` | 权限不足时是否发送提示。关闭后静默忽略无权限操作。 |

配置实现要求：

- 配置项应优先使用 AstrBot 插件配置机制暴露，避免要求用户手改源码。
- 运行时配置变更是否热更新，以 AstrBot 插件机制为准；无法热更新时需在 README 中说明需要重载插件或重启 AstrBot。
- 高危能力默认关闭，包括跨群写入、清空、导入、重建等。
- 所有数量限制应在写入前校验，并返回清晰错误提示。
- 配置默认值应尽量接近 XQA 原始体验，但优先保证安全和可控。

## 11. 安全与风控

### 11.1 防炸群

- 对自动回复增加冷却时间。
- 禁止或限制空匹配、泛匹配问题。
- 默认不允许普通成员设置公共问答。
- 提醒用户不要在同一群放置多个自动问答 Bot。

### 11.2 正则安全

- 正则异常不应导致插件崩溃。
- 危险正则应被拒绝或设置超时。
- 匹配逻辑应捕获异常并继续处理其他规则。

### 11.3 内容安全

- 首版可不内置敏感词库，但应预留过滤接口。
- 对过长回答、过多随机候选、过多问答数量设置限制。

### 11.4 数据安全

- 清空、导入、重建等高危命令需限制到超级管理员。
- 可考虑增加二次确认机制。

## 14. 验收标准

### 14.1 设置与触发

- 发送 `我问菜单你答今天吃黄焖鸡` 后，同一用户发送 `菜单`，Bot 回复 `今天吃黄焖鸡`。
- 其他用户发送 `菜单` 不触发该个人问答。
- 管理员发送 `有人问群规你答禁止刷屏` 后，本群任意用户发送 `群规`，Bot 回复 `禁止刷屏`。
- 普通成员发送 `有人问群规你答xxx` 时，Bot 拒绝并提示权限不足。

### 14.2 匹配优先级

- 同一群中个人问答和公共问答问题相同，触发者优先收到自己的个人问答。
- 完全匹配优先于正则匹配。
- 后设置的正则问题优先于早设置的正则问题。

### 14.3 正则与回流

- 设置 `我问你好(.*)你答你也好$1` 后，发送 `你好呀`，Bot 回复 `你也好呀`。
- 设置非法正则不应导致插件崩溃。
- 设置明显空匹配问题应被拒绝。

### 14.4 随机回答

- 设置 `有人问吃什么你答火锅#烧烤` 后，多次触发只会返回 `火锅` 或 `烧烤`。
- 设置 `我问井号你答A\#B` 后，触发应回复 `A#B`。

### 14.5 管理命令

- `看看我问` 能列出当前用户在当前群设置的问题。
- `看看有人问` 能列出当前群公共问题。
- `不要回答菜单` 能删除当前用户自己的个人问答。
- 管理员能删除公共问答，普通成员不能删除公共问答。

## 15. 风险与待确认问题

- AstrBot 不同平台适配器的消息链、图片、At 表示方式是否统一，需要实现阶段确认。
- AstrBot 插件数据目录与配置读取最佳实践，需要实现阶段按当前 AstrBot 版本确认。
- 正则超时在 Python 标准 `re` 下不直接支持，可能需要采用第三方 `regex`、异步超时隔离，或先做保守规则限制。
- 是否首版支持图片保存，需要根据 AstrBot 消息链下载能力确认。
- 是否需要兼容旧 XQA 数据导入，需要用户确认数据迁移需求。
